# Google: Core Style

Source: https://developers.google.com/style
License: CC BY 4.0
Last refreshed: 2026-04-29

## Voice and tone

- Be conversational and friendly without being frivolous.
- Write for a global audience.

## Person

- Use second person ("you") rather than first-person plural ("we").
- Address the reader directly.

## Voice and tense

- Use active voice. Make clear who is performing the action.
- Use present tense for timeless documentation.

## Sentence structure

- Put conditions before instructions, not after.
- Example: "On Linux, run X" not "Run X if you're on Linux."

## Contractions

- Contractions are acceptable to maintain a conversational tone.

## Headings and capitalization

- Use sentence case for document titles and section headings.
- Use descriptive headings that work as scannable signposts.

## Punctuation

- Use serial (Oxford) commas.
- Use standard American spelling and punctuation.

## Future features

- Don't document features that don't exist yet.
- Avoid "soon", "in a future release", "we plan to", "coming", "will be supported".
- Exception: descriptive future tense for runtime behavior is acceptable ("the function will return X").

## Global audience

- Avoid idioms, regional expressions, and culturally narrow references.
- Use unambiguous date formatting: ISO format (YYYY-MM-DD) or spelled-out month (January 5, 2026).
