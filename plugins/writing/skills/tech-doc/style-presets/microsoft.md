# Microsoft Writing Style Guide (curated subset)

Source: https://learn.microsoft.com/en-us/style-guide/welcome/. This is a curated subset for use by the tech-doc skill, not the full guide. When in doubt, defer to the source.

## Brand voice

- Warm and relaxed, crisp and clear, ready to lend a hand.
- Project friendliness without being frivolous.

## Top 10 prescriptive rules

1. **Bigger ideas, fewer words.** Shorter is always better.
2. **Write like you speak.** Read text aloud. Avoid jargon and overly complex language.
3. **Use contractions** (it's, you'll, you're, we're, let's) to project friendliness.
4. **Get to the point fast.** Lead with the most important. Front-load keywords for scanning.
5. **Be brief.** Give just enough to make decisions confidently. Prune every excess word.
6. **Sentence-case capitalization.** Capitalize only the first word and proper nouns. Never use Title Case.
7. **Skip end punctuation** on titles, headings, subheadings, UI titles, and list items of three or fewer words.
8. **Use the Oxford comma** in lists of three or more items.
9. **One space** after periods, question marks, colons. No spaces around dashes.
10. **Revise weak writing.** Start statements with verbs. Edit out "you can" and "there is, there are, there were."

## Person and voice

- Use second person ("you").
- Use active voice. Make the actor explicit.

## Tense

- Use present tense for current capabilities and behaviors.
- Avoid future tense scaffolding ("will be able to", "is going to").

## Headings

- Sentence case.
- No end punctuation on headings ≤3 words.
- Front-load keywords.

## Lists and procedures

- Numbered lists for sequences.
- Bulleted lists for unordered items.
- Each list item should start with a verb where possible.

## Accessibility

- Alt text on all meaningful images.
- Descriptive link text.
- Color-independent instructions.

## Bias-free communication

- Avoid gendered, racial, ableist, or culturally narrow language.
- Use blocklist replacements (master/slave to primary/secondary; blacklist/whitelist to blocklist/allowlist; sanity check to validation check).

## Global communications

- Write for an international audience.
- Avoid idioms, regional expressions, sports metaphors, cultural references.
- Use simple constructions. Short sentences.

## Code formatting

- Code in code font (backticks in prose).
- UI elements in bold.
- Variable placeholders in italics or angle brackets per project convention.
