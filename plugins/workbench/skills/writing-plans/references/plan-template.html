Implementation Plan

Comment threads on task cards

Date: 2026-05-08 Owner: Pascal Goellner Active Source spec

Summary

Goal
Add inline comment threads to task cards so collaborators can discuss work without leaving the board view.
Architecture
  • New comments table joined to tasks by task_id.
  • Server-side rendered list with progressive enhancement for inline reply.
  • Read endpoint piggybacks on the existing task fetch to avoid a second round trip.
Tech stack
  • Backend: Node 20, Fastify, Postgres 16, Knex migrations.
  • Frontend: TypeScript, Preact, Tailwind, Vitest.
  • Testing: Vitest unit, Playwright e2e.
Out of scope
Notifications, reactions, threaded replies (single-level only for v1).

Tasks

Task 01

Create the comments table and migration

  • Createapps/api/src/db/migrations/20260508_comments.sql
  • Modifyapps/api/src/db/schema.ts

  1. Write the migration that defines the comments table.

    CREATE TABLE comments ( id uuid PRIMARY KEY DEFAULT gen_random_uuid(), task_id uuid NOT NULL REFERENCES tasks(id) ON DELETE CASCADE, author_id uuid NOT NULL REFERENCES users(id), body text NOT NULL, created_at timestamptz NOT NULL DEFAULT now() ); CREATE INDEX comments_task_idx ON comments(task_id, created_at);

  2. Run the migration locally and confirm the table exists.

    npm run db:migrate && psql $DATABASE_URL -c "\d comments"
    Expected: table description prints with six columns and one index.

  3. Commit when the migration applies cleanly on a fresh database.

    git add apps/api/src/db/migrations apps/api/src/db/schema.ts && git commit
Task 02

Add the POST /tasks/:id/comments route

  • Createapps/api/src/routes/comments.ts
  • Createapps/api/src/services/comments.ts
  • Modifyapps/api/src/app.ts

  1. Implement the service that inserts a comment and returns the created row.

    export async function createComment(input: NewComment) { const [row] = await db("comments").insert(input).returning("*"); return row; }

  2. Register the route on the Fastify app and validate the body with the existing zod schema.

    app.register(tasksRoutes); app.register(tasksRoutes); app.register(commentsRoutes);

  3. Run the route test suite and confirm a 201 response body matches the schema.

    npm test -- routes/comments
    Expected: 4 tests passing, no skipped, status 201 with id and created_at.
Task 03

Render CommentThread inside TaskCard

  • Createapps/web/src/components/CommentThread.tsx
  • Createapps/web/src/components/CommentForm.tsx
  • Modifyapps/web/src/components/TaskCard.tsx

  1. Build the read-only CommentThread component that renders the list of comments passed in via props.

  2. Add CommentForm with optimistic insert and error rollback on failure.

  3. Wire both into TaskCard behind the FEATURE_TASK_COMMENTS flag.

    npm run test:e2e -- task-comments
    Expected: e2e flow opens a card, posts a comment, sees it appear without reload.
